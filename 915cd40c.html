Explain localhost and 127.0.0.1

Posted on 2023-07-14 In Network

Explain Localhost and 127.0.0.1 In a Easy Way

localhost Makes Life Easier for Beginners

Let’s start with an example:

when you write a server application, and you don’t have a remote server running yet, you might ask yourself: “ How can I test my server app? “. That is the moment when localhost comes to the rescue, because you don’t need a remote server yet, to test your application.

How Does localhost Work ?

When you try to call a website, your computer tries to connect to a remote computer (aka server) somewhere on the internet. It sends a request and, as a result, receives a response with some content.

When you call localhost (or 127.0.0.1), you send a request to your own device. Your computer recognizes the address (localhost or 127.0.0.1), and instead of sending it out over the internet, it returns the request back through a loopback mechanism. Keeping the server app example in mind, now your server (on the same machine) can receive the request and send a response with some content.

The requests/responses will never leave your computer. In other words: By calling localhost, your computer always talks with itself.

The Difference Between 127.0.0.1 and localhost

IP address representation and Special IP address

The IP address consists of two parts, net-id, and host-id, namely the network and host numbers. net-id: Indicates the network number where the IP address is located. host-id: Indicates a host number in the network where the IP address is located.

IP-address ::=  { <Network-number>, <Host-number> }

Special IP addresses are used to do some special things. The following special IP addresses are defined in RFC1700.

  1. {0,0}: Both the network number and the host number are 0, which means “this host on this network”, and can only be used as the source address.

  2. {127,}: Any IP address whose network number is 127. They are all internal host loopback addresses (loopback), which can never appear in the network outside the host.

Answers to questions

Compared to 127.0.0.1, localhost has more meaning. localhost is a domain name, not an IP address. We often think of localhost and 127.0.0.1 as the same because most of the computers we use say that localhost points to the address 127.0.0.1. In the ubuntu system, the /etc/hosts file will have the following content:

127.0.0.1   localhost

However, configuring the host file can easily link localhost with a different IP address, as this file contains the mappings of IP addresses to hostnames.

Let me explain a little bit of Bind

Networking:

Networking refers to the practice of connecting multiple computers and devices to create a network. It enables communication and data exchange between these devices, allowing them to share resources and information. Networks can be classified into different types, such as Local Area Networks (LANs) or Wide Area Networks (WANs), depending on their geographical scope.

In computer networks, devices are identified by unique addresses called IP (Internet Protocol) addresses. IP addresses can be either IPv4 (e.g., 192.168.0.1) or IPv6 (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334) format.

Bind (Berkeley Internet Name Domain):

BIND is the most widely used DNS (Domain Name System) software on the internet. DNS translates domain names (e.g., example.com) into IP addresses that computers can understand. It enables us to access websites, send emails, and perform various internet-related activities using easy-to-remember domain names instead of complex IP addresses.

BIND consists of two main components: a DNS server (named) and a DNS resolver (named-resolver). The DNS server handles domain name queries and provides authoritative answers, while the resolver processes queries from clients and interacts with the DNS server to resolve domain names.

Configuration files, such as named.conf, are used to specify how BIND operates. These files contain various directives and options to control the DNS server’s behavior, including the ‘listen-on’ directive you mentioned earlier.

The ‘listen-on’ directive:

The ‘listen-on’ directive in BIND’s named.conf file specifies the IP address and port number on which the DNS server should listen for incoming queries. By default, it restricts BIND to listen on the loopback address (127.0.0.1) on port 53, which means it will only accept queries from the local machine.

When you replaced ‘127.0.0.1’ with localhost in the ‘listen-on’ directive, BIND interpreted it as the loopback address and allowed external queries. This behavior occurred because localhost typically resolves to the loopback address in most systems.

It’s important to note that BIND’s behavior may vary based on its version, configuration, and platform-specific settings. It’s always recommended to consult BIND’s official documentation or seek assistance from experienced administrators when working with complex BIND configurations.
